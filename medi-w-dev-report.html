프로젝트 개요
MEDI-W 플랫폼의 어드민 시스템 개요 및 현재 개발 상태

MEDI-W는 해외 환자를 국내 병원·의료 기관과 연결하는 글로벌 의료 허브 플랫폼입니다. 이 문서는 MEDI-W Admin(관리자 시스템)의 정적 HTML 시안을 기반으로, 실제 개발에 필요한 전체 기능 명세와 기술 사양을 정리합니다.

현재 상태: 디자인 시안(정적 HTML/CSS) 완성. 백엔드·동적 기능 미구현. 본 문서는 실제 애플리케이션 개발을 위한 명세서입니다.

API 명세
개발 필요 REST API 전체 목록
Base URL: https://api.medi-w.com/v1 — 모든 API는 JWT Bearer Token 인증 필요 (로그인 API 제외)
🔐 인증 (Auth)
POST/auth/login이메일/비밀번호 로그인 → Access Token + Refresh Token 발급
POST/auth/logout로그아웃 — Refresh Token 무효화
POST/auth/refreshAccess Token 갱신
GET/auth/me현재 로그인 어드민 정보 조회
PATCH/auth/password비밀번호 변경
🏥 병원 (Hospitals)
GET/hospitals병원 목록 조회 (페이지네이션, 검색, 필터: region, specialty, status)
GET/hospitals/:id병원 상세 조회
POST/hospitals신규 병원 등록 (multipart/form-data, 이미지 첨부)
PUT/hospitals/:id병원 정보 전체 수정
PATCH/hospitals/:id/status병원 상태 변경 (active / inactive / pending)
PATCH/hospitals/order병원 노출 순서 일괄 업데이트 (드래그 결과)
DELETE/hospitals/:id병원 삭제 (소프트 삭제)
GET/hospitals/search지도 기반 병원 검색 (lat, lng, radius, keyword)
🧑‍⚕️ 환자 (Patients)
GET/patients환자 목록 조회 (필터: country, status, hospital, dateRange)
GET/patients/:id환자 상세 조회 (기본 정보 + 상담 이력)
POST/patients신규 환자 등록
PATCH/patients/:id환자 정보 수정
DELETE/patients/:id환자 삭제 (소프트 삭제)
GET/patients/:id/consultations환자별 상담 이력 조회
💬 상담/예약 (Consultations)
GET/consultations상담 목록 조회 (필터: status, hospital, patient, dateRange)
GET/consultations/counts칸반 컬럼별 건수 (PENDING/IN_PROGRESS/CONFIRMED/COMPLETED)
GET/consultations/:id상담 상세 조회
POST/consultations신규 상담 등록
PATCH/consultations/:id/status상담 상태 변경 (칸반 드래그 시 호출)
PATCH/consultations/:id/assign병원 배정 (hospitalId 전달)
🤝 에이전시 (Agencies)
GET/agencies에이전시 목록 (필터: country, status, sort)
GET/agencies/:id에이전시 상세 (기본 정보 + 실적)
POST/agencies에이전시 등록
PATCH/agencies/:id에이전시 정보 수정
PATCH/agencies/:id/status에이전시 상태 변경
GET/agencies/:id/settlements에이전시 수수료 정산 내역
📊 통계 (Statistics)
GET/stats/summary대시보드 요약 (병원 수, 환자 수, 상담 수, 매출)
GET/stats/operations운영 통계 (기간별 신규 병원/환자/상담 추이)
GET/stats/patients환자 분석 (국가별, 연령별, 성별 분포)
GET/stats/revenue수익 분석 (매출 추이, 수수료 분포)
GET/stats/export통계 데이터 CSV/Excel 다운로드
🔔 알림 (Notifications)
GET/notifications알림 목록 (필터: type, unread)
PATCH/notifications/:id/read개별 알림 읽음 처리
PATCH/notifications/read-all전체 알림 읽음 처리
GET/notifications/stream실시간 알림 SSE(Server-Sent Events) 스트림
⚙️ 설정 (Settings)
GET/settings전체 시스템 설정 조회
PATCH/settings/general일반 설정 저장 (플랫폼명, 이메일 등)
PATCH/settings/notifications알림 채널 설정 저장
GET/settings/permissions역할별 권한 목록 조회
PATCH/settings/permissions권한 설정 저장
POST/settings/backup데이터 백업 실행
GET/settings/audit-logs감사 로그 조회
