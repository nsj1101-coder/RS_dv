- CTA 버튼 — 환자/에이전시: "병원 검색 · 접수" → hospitals.html / 의료인: "프로그램 지원" → programs.html

- 알림 리스트 (타입별 다른 내용)

**체크 사항:**

- [ ] CTA 버튼 (환자/에이전시) → hospitals.html 이동 확인
- [ ] CTA 버튼 (의료인) → programs.html 이동 확인
- [ ] 사이드바 메뉴 → 각 페이지 이동 확인
- [ ] 로그아웃 → login.html 이동
- [ ] 모바일 햄버거 메뉴 토글

### U-2. 내 정보 / 프로필 (`profile.html`)
**노출 정보:**
- 타입별 프로필 폼:
  - **환자/보호자 (A):** 환자 본인 여부, 보호자 관계, 긴급 연락처, 방문 목적, 방문 예정 국가, 여권번호(선택)
  - **의료인 (B):** 이름, 국적, 거주국가, 근무병원, 병원주소, 직위, 전공분야, 임상경력, 근무기간, 의료면허번호, 면허 발급국가, 면허기관, 관심 연수과, 희망 연수기간, 희망 시작시기, 파일 업로드(면허증/CV/재직증명/추천서)
  - **에이전시 (C):** 회사명(한/영), 사업자등록번호, 설립국가, 회사주소, 웹사이트, 대표자명, 담당자 정보, WhatsApp/Telegram, 사업유형, 연간 환자 송출 규모, 주요 송출 국가, 주요 협력 병원, 파일 업로드(사업자등록증/회사소개서/로고), 정산계좌(은행/계좌/SWIFT/계좌주)
- 프로필 완성도 프로그레스 바 (상단 고정)
- 필수 필드 빨간색 * 표시

**체크 사항:**
- [ ] 타입 스위처 전환 시 폼 내용 변경
- [ ] 프로필 완성도 계산 (입력 필드 채울수록 % 증가)
- [ ] 프로그레스 바 스크롤 시 상단 고정 (sticky)
- [ ] 파일 업로드 영역 클릭 → 파일 선택 다이얼로그
- [ ] "저장" 버튼 → 토스트 알림
- [ ] 필수 값 * 빨간색 표시 확인

### U-3. 환자 관리 리스트 (`patient-list.html`)

- 상태 필터 탭 (전체, 작성 중, 접수 완료, 검토 중, 자료 요청, 병원 승인, 최종 확인 대기, 최종 확정, 철회)
- 검색 바
- 환자 리스트 테이블 (이름, 병원, 진료과, 상태, 진행 단계, 등록일)
- "새 환자 등록" 버튼

**체크 사항:**
- [ ] 상태 필터 탭 클릭 시 리스트 필터링

- [ ] "새 환자 등록" → patient-intake.html 이동
- [ ] Step 진행 표시 (Step 1/2 또는 완료)

### U-4. 환자 상세 (`patient-detail.html`)
**노출 정보:**
- 환자 헤더 (이름, 상태 뱃지, 병원, 진료과)
- 승인 플로우 타임라인 (5단계: 접수 → 검토 → (자료요청) → 병원승인 → 최종확인 → 확정)
- **추가 자료 요청 알림** (자료 요청 상태일 때 표시)
  - 요청 사유, 요청 항목 리스트, 업로드 영역, 제출 버튼
- **최종 확인 대기** (병원 승인 후 환자/에이전시 확인 단계)
  - 승인, 철회 버튼
- 정보 아코디언 카드:
  - 기본 정보 (이름, 국적, 연락처 등)
  - 의료 정보 (진료과, 증상, 진단, 약물, 병력)
  - 의료 기록 (업로드된 파일 리스트)
  - 방문 계획 (방문 시기, 컨시어지 서비스)

**체크 사항:**
- [ ] `?id=1` → 자료 요청 상태 (추가 자료 요청 알림 노출)
- [ ] `?id=3` → 최종 확인 대기 상태 (승인/철회 버튼 노출)
- [ ] `?id=17` → 철회 상태
- [ ] 아코디언 카드 펼침/접힘 동작
- [ ] 추가 자료 업로드 후 제출 동작
- [ ] "승인" / "철회" 버튼 클릭 시 상태 변경 확인
- [ ] 뒤로가기 → patient-list.html 이동

### U-5. 환자 접수 2단계 (`patient-intake.html`)
**노출 정보:**
- **Step 1 — 기본 정보:**
  - 선택된 병원/진료과 표시 영역 (hospitals.html에서 넘어올 경우)
  - 환자 이름, 성별, 생년월일, 국적, 전화번호, 이메일
  - 희망 진료과 (칩 선택)
  - 주요 증상 (텍스트 에어리어)
- **Step 2 — 상세 의료정보:**
  - 현지 진단명, 진단 날짜, 진단 병원
  - 병력 / 수술 이력
  - 현재 복용 약물
  - 의료 기록 업로드 (Diagnosis Report, CT, MRI, X-ray, Blood Test, Pathology, Prescription, 기타)
  - 방문 일정 (방문 목적, 희망 시기, 체류 기간)
  - 컨시어지 서비스 (공항픽업, 차량, 숙박, 통역 토글)

**체크 사항:**
- [ ] `?hospital=서울아산병원&depts=심장내과||신경과` 파라미터 시 선택 병원/진료과 자동 표시
- [ ] Step 1 → Step 2 전환 애니메이션
- [ ] Step 2 → Step 1 뒤로가기
- [ ] 진료과 칩 다중 선택
- [ ] 파일 업로드 슬롯별 개별 파일 선택
- [ ] 컨시어지 토글 ON/OFF
- [ ] "접수하기" → 완료 알림 후 patient-list.html 이동
- [ ] "임시 저장" → patient-list.html에서 "작성 중" 상태로 노출

### U-6. 병원 검색 (`hospitals.html`)
**노출 정보:**
- 3컬럼 레이아웃:

  - **병원 상세:** 외관 이미지, 병원명, 주소, WhatsApp/Email, JCI/보건복지부 인증 뱃지, 설명, 지도, 진료과 목록
- 진료과 행 클릭 → **교수진 카드 리스트 펼침** (아코디언)
- 교수 카드 클릭 → **교수 상세 모달** (이름, 직위, 경력, 학력, 세부전공, 주요 질환, 연구실적, 진료 언어)
- 진료과 체크박스 선택 → 하단 "의료 접수" 플로팅 바

**체크 사항:**
- [ ] 병원 목록 12개 노출 확인
- [ ] 검색 (병원명, 진료과, 병명, 교수명) 실시간 필터
- [ ] 필터 칩/체크박스 클릭 시 목록 필터링
- [ ] "초기화" 버튼 → 모든 필터 해제
- [ ] 병원 카드 클릭 → 우측 상세 패널 전환
- [ ] 진료과 행 클릭 → 교수진 카드 펼침/접힘
- [ ] 교수 카드 클릭 → 모달 프로필 (ESC/배경 클릭으로 닫기)
- [ ] 진료과 체크박스 선택 → 하단 플로팅 바 "N개 진료과 선택됨" + "의료 접수" 버튼
- [ ] "의료 접수" → patient-intake.html?hospital=OOO&depts=OOO 이동 (선택 정보 전달)

## 3. 테스트 플로우

### Flow 1: 신규 가입 → 대시보드 진입
```
index.html → "Get Started" 클릭
→ signup.html → Step 1~4 순서대로 입력/다음
→ signup-success.html → "Go to Dashboard" 클릭
→ login.html → 이메일/비밀번호 입력 → "Sign In"
→ dashboard.html (대시보드 노출)
```

### Flow 2: 프로필 작성 (에이전시 기준)
```
dashboard.html → 사이드바 "내 정보" 클릭
→ profile.html → 타입 스위처에서 "에이전시" 선택
→ 회사명, 사업자번호, 담당자 정보 등 입력
→ 파일 업로드 (사업자등록증, 회사소개서, 로고)
→ "저장" 클릭 → 완성도 % 증가 확인
```

### Flow 3: 병원 검색 → 진료과 선택 → 환자 접수
```
dashboard.html → 사이드바 "병원 검색" 클릭
→ hospitals.html → 좌측 필터에서 "심장내과" 클릭
→ 목록에서 "서울아산병원" 클릭 (우측 상세 표시)
→ 진료과 "심장내과 (Cardiology)" 행 클릭 → 교수진 카드 펼침
→ "박승정" 교수 클릭 → 모달 프로필 확인 → X 닫기
→ 체크박스로 진료과 선택 → 하단 "의료 접수" 클릭
→ patient-intake.html (선택된 병원/진료과 자동 표시)
→ Step 1 입력 → "다음" → Step 2 입력 → "접수하기"
→ patient-list.html (신규 환자 행 확인)
```

### Flow 4: 환자 리스트 → 상세 확인 → 추가 자료 제출
```
patient-list.html → 상태 필터 "자료 요청" 클릭
→ 해당 환자 행 클릭
→ patient-detail.html?id=1 → 추가 자료 요청 알림 확인
→ 요청 항목 확인 (예: MRI 최신본, 혈액검사 결과)
→ 파일 업로드 → "자료 제출" 클릭
```

### Flow 5: 양측 승인 플로우
```
patient-detail.html?id=3 → 최종 확인 대기 상태
→ "승인" 또는 "철회" 버튼 확인
→ "승인" 클릭 → 최종 확정 상태로 전환
(또는 "철회" 클릭 → 철회 사유 입력 → 철회 상태로 전환)
```

### Flow 6: 어드민 — 병원 등록 및 관리
```
admin.html → 사이드바 "병원 관리" 클릭
→ admin-hospitals.html → "+ 병원 등록" 클릭
→ hospital-register.html → 7개 섹션 폼 입력
→ 진료과 3개 추가 (각각 의사 이름 입력)
→ JCI 토글 ON, 파일 업로드
→ "병원 등록" 클릭 → 검증 → 완료
→ admin-hospitals.html → 등록된 병원 행 확인
→ "상세 →" 클릭 → hospital-detail.html 확인
```

### Flow 7: 어드민 — 승인 처리
```
admin.html → 사이드바 "승인 관리" 클릭
→ admin-approvals.html → "병원 검토 대기" 필터 클릭
→ 승인 대기 카드 확인 → "승인" 버튼 클릭
→ "환자 확인 대기" 필터 → "확인 요청" 버튼 클릭
```

## 4. 상태값 정의 (환자 케이스)

| 상태 | 영문 | 색상 | 설명 |
|------|------|------|------|
| 작성 중 | draft | 회색 | Step 1 또는 Step 2 미완료 |
| 접수 완료 | submitted | 파랑 | 모든 정보 입력 완료 |
| 검토 중 | reviewing | 주황 | 병원에서 서류 검토 중 |

| 병원 승인 | hospital_approved | 보라 | 병원에서 승인 완료 |
| 최종 확인 대기 | pending_confirm | 주황 | 환자/에이전시 최종 확인 대기 |
| 최종 확정 | confirmed | 초록 | 양측 모두 승인 완료 |
| 철회 | withdrawn | 회색 | 환자/에이전시가 철회 |
